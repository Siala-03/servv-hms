## Booking.com Live Integration

This project includes a live Booking.com sync path in the backend at:

- `POST /api/channels/booking-com/sync`

Required backend environment variables:

- `BOOKING_COM_HOTEL_ID`
- `BOOKING_COM_SYNC_URL`
- `BOOKING_COM_TOKEN` **or** both `BOOKING_COM_USERNAME` + `BOOKING_COM_PASSWORD`

Behavior:

- Pulls current room inventory/rate shape from your HMS (`rooms` table)
- Sends payload to Booking.com live endpoint
- Records sync result in `channel_sync_results`

## SPA Routing (No 404 On Refresh)

This app uses client-side routing with React Router. A direct refresh on pages like `/reservations` or `/guests` must return `index.html` from hosting.

- Vercel: already configured via [vercel.json](vercel.json).
- Other hosts: add an SPA fallback rewrite so all non-file routes map to `index.html`.
